# Lint gate E1 baseline — grandfathered oversized .c files (app/ + config/).
#
# check-file-size-ceiling (tools/scripts/check_file_size_ceiling.sh) is an
# ENFORCED RATCHET (not advisory): it flags any app/**/*.c or config/src/*.c
# whose line count exceeds the ceiling (800). Files pre-dating the gate are
# recorded here so the ratchet does not regress. Each entry is
# '<relative path> <LOC>'. The gate FAILS (nonzero exit) when:
#   - a NEW file (not listed here) exceeds the ceiling, OR
#   - a listed file grows ABOVE its recorded baseline LOC, OR
#   - a listed file is missing from the tree while the gate still expects
#     it (removed/renamed without updating this file).
# Shrinking a file below the ceiling lets you delete its baseline line —
# the gate suggests this automatically when a baselined file's live LOC
# is below its recorded number. Re-baselining to a LOWER count for a file
# that shrank but is still over the ceiling is always welcome; raising a
# recorded count requires an ADR (this is a ratchet, not a target).
#
# Format: <relative-path> <max-loc>   (lines starting with # are comments)
# Sorted by path.
