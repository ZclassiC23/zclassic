# Lint gate E1 (WARN tier) baseline — grandfathered oversized .c files
# under lib/ (excl. lib/test/, which is fixtures/registrations and
# legitimately huge), domain/, and src/ (the binary's composition
# entrypoint, e.g. main.c).
#
# check-file-size-ceiling (tools/scripts/check_file_size_ceiling.sh) treats
# this tier as WARN-only: it never fails the build. lib/ and domain/ are
# primitives and pure-domain code, not the eight app-shape surfaces E1's
# ENFORCED tier was written to police, so this baseline exists to make
# growth visible, not to gate it. Each entry is '<relative path> <LOC>'.
# The gate prints when:
#   - a NEW file (not listed here) exceeds the ceiling (800), OR
#   - a listed file grows ABOVE its recorded baseline LOC.
# Shrinking a file below the ceiling lets you delete its baseline line.
#
# Straight-line generated/tabular crypto & math code does not belong here
# at all — it is skipped entirely via LIB_ALLOWLIST in the gate script
# (lib/chain/src/sha3_windows.c, lib/sapling/src/bn254.c,
# lib/sapling/src/bls12_381.c, domain/wallet/src/mnemonic.c).
#
# Format: <relative-path> <max-loc>   (lines starting with # are comments)
# Sorted by path.
