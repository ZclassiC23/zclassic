# check-long-functions (tools/scripts/check_long_functions.sh) is an
# ENFORCED RATCHET (not advisory): it flags any function in this tier whose
# body exceeds the cap (500 lines). Functions pre-dating this baseline's
# introduction are recorded here so the ratchet does not regress. Each
# entry is '<relative path> <function-name> <max-lines>'. The gate FAILS

# the gate suggests this automatically when a baselined function's live
# length is below its recorded number. Re-baselining to a LOWER count for
# a function that shrank but is still over the cap is always welcome;
# raising a recorded count requires an ADR (this is a ratchet, not a
# target).
#
# Format: <relative-path> <function-name> <max-lines>   (# = comment)
