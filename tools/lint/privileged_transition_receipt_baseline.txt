# check_privileged_transition_receipt.sh baseline — Law-7 dispositions.
#
# Every native command leaf whose spec is ZCL_COMMAND_AUTH_OWNER AND effect
# ZCL_COMMAND_EFFECT_MUTATE or ZCL_COMMAND_EFFECT_DESTRUCTIVE is a candidate
# privileged transition. Each such leaf MUST have a disposition here; a new
# owner-mutating leaf with none FAILS the gate, forcing a conscious Law-7
# review. RATCHET: this file only grows when a genuinely new owner-mutating
# leaf is added — and it should grow with `receipt:` far more often than
# `exempt:` from now on.
#
# Line format (one leaf per line; `#` comments / blank lines ignored):
#   <leaf.path>  receipt:<relative_file>   # binds an authority_receipt verifier
#   <leaf.path>  exempt:<one-line reason>  # not an artifact-install transition
#
# For a `receipt:<file>` line the gate additionally asserts <file> contains a
# call matching authority_receipt_.*_available( OR the pre-generalized
# consensus_state_replay_receipt_authority_available( — a positive check that
# the wired consumer actually gates on the contract.
#
# Today's universe (25 leaves — the initial baseline, all `exempt`). The first
# real privileged artifact-install transition surfaced as an OWNER leaf (bundle
# ACTIVATE, hot-swap Phase-3 reopen, `make deploy` generation publish, App
# cartridge activation) lands here as `receipt:<handler_file>`.
