# ZClassic23 Full Code-Review Synthesis

**Scope:** 32 modules, ~237k LOC single C23 binary (ZClassic full node). **Dimensions:** DRY, Navigability (AI-friendliness), Clarity, API quality, Correctness/quality, Tested, Validation+logging. **Verdict:** `solid-with-gaps`.

---

## Executive Summary

This is a high-quality, defensively-engineered codebase that is genuinely competitive — but not yet at the "$1M sync-or-die" bar. The reducer fold engine, the no-silent-halt discipline, the CSR single-writer chokepoint, the self-heal honest-witness termination, and the crypto/sapling soundness defenses are best-in-class. The gaps cluster on the highest-weighted surfaces: boot/loader/snapshot/net correctness, wallet write paths, and **test coverage of exactly the code that wins or loses the competition** (gap-fill un-wedge, loader storm-fix, snapshot apply, Sprout PHGR13 verifier). No consensus-rule divergence ships — parity is well-guarded and tighteners are correctly replay-gated — but several boot/net defects can lose a single sync run.

## Cross-Cutting Patterns (highest-leverage fixes)

1. **Copy-pasted trust-boundary parsers** — UTXO chunk parser (2 copies: `snapshot_fetch.c:296` vs `snapshot_sync_service.c:707`), CCoins decoder (3 copies: `utxo_import_pipeline.c:75`, `coins_db.c:98`, `chainstate_legacy_reader.c:174`), snapshot-loader walk (`utxo_snapshot_loader.c:152` vs `192`), MMB peak-merge (`mmb.c:108` vs `328` — drift silently breaks FlyClient proofs), multipack-to-Fr (3 copies in sapling). **Fix:** one shared decoder per wire format.

2. **Duplicated consensus selection logic** — `find_most_work_chain` (`process_block_core.c:23`) vs `active_chain_most_work_candidate` (`chainstate.c:718`); a third selector already diverged and wedged the tip (`process_block_tip_publish.c:70`). **Fix:** one `select_most_work()` helper.

3. **Boot/recovery diagnostics to stdout, not node.log** — flat block-index loader (`block_index_loader.c:237-413`), chain-restore verdicts (`chain_restore_repair.c:341-795`), find_most_work_chain STUCK line (`process_block_core.c:101`), snapshot dispatcher rejects (`msgprocessor_snapshot.c:1136`), explorer stats (`explorer_stats_view.c:382`), sapling params (`params_init.c:196` — also corrupts -mcp stdio). **Fix:** convert to LOG_*; never stdout from lib/service code.

4. **Unchecked snprintf truncation on path/SQL/body** — `boot_snapshot_import.c:202` (ATTACH SQL), `sync_controller.c:207`, `explorer_pages_view.c:82`, `client.c:198`, `tools/mcp/rpc_client.c:139` (OOB stack read). **Fix:** check `n >= sizeof(buf)`; bind paths in SQL; add a lint.

5. **Wall-clock timing (gettimeofday) on elapsed/deadline paths** — evades Gate #19. `snapshot_sync_service.c:98` (aborts fast-sync), `rpc_timeout.c:53`, `consensus_reject_index.c:55`, `msgprocessor.c:586`, `p2p_game.c:13`, `db_txn.c:36`, `mmb_leaf_store.c:174`. **Fix:** platform_time_monotonic_us(); ban gettimeofday in the gate.

6. **AR write-lifecycle misuse / bypass** — read-macro on writes (`sync_controller_blocks.c:329`, `utxo_recovery_backfill.c:113`, `utxo_import_pipeline.c:62`, `block_index_loader.c:527`), raw exec writes (`boot_services.c:693`, `src/main.c:1106` ungated because src/ is outside the lint root). **Fix:** AR_STEP_WRITE / AR_ADHOC_SAVE; add `src` to lint roots.

7. **Unchecked fixed-blob column memcpy (OOB read)** — `swap_contract.c:126`, `file_offer.c:84` (peer-gossip rows), `znam.c:160`, `zmsg.c:84`. **Fix:** mechanical substitution to AR_READ_BLOB.

8. **Dead code / stale contracts on load-bearing paths** — `boot.c:1186` (2739-line app_init), `msgprocessor_snapshot.c:749` (927-line dispatcher), dead `return true;` after LOG_FAIL, rotted line refs, header contracts contradicting code (`db_txn.h:91`, `connect_block.c:188`, `boot_refold_staged.c:203`, `snapshot_verify.c:506`). **Fix:** decompose god-functions; reconcile contracts.

## What's Excellent / Keep Doing

- **The 8 reducer fold stages (jobs-stages) are the gold standard** — every reject carries height + txid/vout + the exact zclassicd reject string; every JOB_IDLE/BLOCKED is a named cause; tighteners are default-OFF and replay-gated per the h=478544 doctrine; consensus alignment is cited file:line. This is what the rest of the codebase should imitate.
- **No-silent-halt discipline is real and pervasive** — across svc-evidence, svc-sync-mirror, svc-health-misc, lib-validation, the conditions registry and the loaders, a stall is a named blocker with height/table/reason, not a quiet stop. Validation+logging is the strongest dimension overall.
- **The CSR single-writer chokepoint** (chain_state_service.c) — every tip mutation validated under one mutex, typed csr_result, nothing published if any cross-check fails, with the 1.3M-UTXO-loss incident documented inline.
- **Self-heal honest-witness termination** (app/conditions) — every remedy verifies the symptom moved (tip/cursor/H* advanced), bounded by max_attempts or cooldown re-arms; structurally prevents result=ok-while-stuck.
- **Crypto/sapling soundness defenses** — Groth16 prime-order subgroup membership (closes torsion forgery), constant-time scalar mul, SHA-512 param-hash refuse-to-start, fail-closed RNG, constant-time auth/AEAD compares, secret memory_cleanse on every path.
- **Trust-boundary length-bounding before allocation** in lib-net-core and lib-misc transaction/block parsers (count bounded vs consensus max AND stream_remaining before the calloc) — the classic huge-count OOM DoS is defeated with overflow reasoning inline.
- **Strong module decomposition for AI navigability** in the controllers, jobs, and validation units — one responsibility per file with internal headers pointing at where things live (the boot.c and msgprocessor_snapshot.c god-functions are the exceptions, not the rule).
- **The destructive-wipe and snapshot-trust choke points** (utxo_recovery_wipe, snapshot anchor-root bind, boot_import_snapshot_db) — policy-gated, row-count-first, fail-closed on every consensus-binding gap, refuse above-checkpoint peer snapshots.
