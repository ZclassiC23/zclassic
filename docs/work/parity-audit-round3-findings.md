## Disposition
1. **Lock-in tests first (safe, no consensus change)** — pin CURRENT behavior for D1–D5 so each gap is visible and a future fix flips a test deliberately. D2 is partially pinned already.
2. **D1 (difficulty precedence) is the cleanest parity-restore** — mainnet byte-identical, so fixing the paren cannot change the mainnet floor. It still alters testnet/regtest → replay-gate against testnet/regtest history before shipping.
3. **D2 (coinbase maturity) is the most consequential live loosening** — a real block-consensus gap on the coins_kv author. Enable the default only after a full-history replay confirms 0 false-rejects (the h=478544 doctrine). It is honest today (default-accept matches no real divergent block).
4. **D3 (BIP34 1-16)** ancient/low; **D4 (time-offset)** soft/peer-dependent; **D5 (mempool finality)** relay-only — all documented, none a mainnet block-consensus fork.
